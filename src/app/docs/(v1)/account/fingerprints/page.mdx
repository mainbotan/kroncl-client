# 1. Ключи доступа

Существует множество ситуаций, в которых необходимо предоставить доступ непосредственно к личному аккаунту (не конкретной организации), при этом не раскрывая учётные данные третьим лицам.

Для этих случаев предусмотрены <span className={styles.underline}>временные ключи</span> доступа к вашему аккаунту.

Работает это по принципу: владелец аккаунта выпускает ключ с ограниченным временем жизни, третьи лица используют полученный ключ <Link className={styles.link} href='/sso/fingerprint'>на странице</Link> авторизации по ключу.

# 1.1 Просмотр ключей

В разделе <Link href='/platform/security' className={styles.link}>«Безопасность»</Link> (левое боковое меню) отображается список уже выпущенных ключей доступа к аккаунту, включая их срок жизни, статус активности, дату выпуска и маску ключа (полный ключ доступен только один раз после создания).

<MDXImageBlock src='/images/docs/account-fingerprints.png' label='Просмотр выпущенных ключей.' />

# 1.2 Создание ключа

Для выпуска нового ключа необходимо указать его срок жизни. Срок жизни позволяет ограничить возможность авторизации по ключу в рамках целевого промежутка времени.

Ключ доступа показывается только один раз после инициализации — скопируйте его и передайте получателю.

<MDXImageBlock src='/images/docs/create-account-fingerprint.png' label='Создание ключа.' />

# 1.3 Деактивация ключа

Для деактивации нажмите на нужный ключ в списке уже выпущенных, после чего подтвердите деактивацию.

После деактивации возобновить действие ключа будет невозможно.

<MDXImageBlock src='/images/docs/deactivate-account-fingerprint.png' label='Деактивация ключа.' />
