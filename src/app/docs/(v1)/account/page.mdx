# 1. Аккаунт

Для начала работы на платформе достаточно <Link className={styles.link} href='/sso/sign_up'>зарегистрировать</Link> в системе аккаунт и подтвердить почту с помощью кода подтверждения.

Личный аккаунт не ограничен одним экземпляром учётной системы. Напоминаем, что один аккаунт может состоять во множестве организаций.

Ваша рабочая почта является основным способом входа в аккаунт. В случае утери пароля доступ к учётной записи можно <Link className={styles.link} href='/sso/recovery'>восстановить</Link> с её помощью.

# 1.1 Методы авторизации

Kroncl предоставляет возможность быстрой авторизации или регистрации в системе с помощью сторонних приложений (Яндекс ID, Google и другие).

Важно понимать: при выборе этих методов вы создаёте или авторизуетесь в аккаунте Kroncl через почту, которая привязана к внешнему провайдеру.

Например, если вы изначально вручную зарегистрировали аккаунт на почту <span className={styles.tag}>user@gmail.com</span>, а затем используете авторизацию через Яндекс ID с почтой <span className={styles.tag}>user@yandex.ru</span> — система распознает новую почту и создаст новый аккаунт.

# 1.2 Подтверждение почты

Важным этапом создания аккаунта является подтверждение почты. Сразу после регистрации мы пришлём код подтверждения на указанную в процессе почту. Этот код понадобится вам один раз — для завершения создания аккаунта.

Как уже было сказано, почта — ваш основной способ доступа к аккаунту. Именно на неё в дальнейшем будут приходить приглашения в организации.

# 1.3 Доступ третьим лицам

В корпоративной среде нередки ситуации, когда требуется временно предоставить доступ к аккаунту третьему лицу, не раскрывая при этом свои учётные данные.

В таких случаях мы настоятельно рекомендуем создать третьему лицу личный аккаунт, а затем пригласить его в целевую организацию через её владельцев.

Однако иногда возникает необходимость дать доступ от первого лица — например, для разового входа без дополнительных манипуляций с приглашениями, во все организации, где вы уже состоите.

В подобных ситуациях мы <span className={styles.underline}>настоятельно не рекомендуем</span> передавать пароль от своего аккаунта.

Используйте специальные ключи с ограниченным временем жизни — они позволяют авторизоваться под вашим аккаунтом третьим лицам. <Link href='/docs/account/fingerprints' className={styles.link}>Подробнее о ключах.</Link>
