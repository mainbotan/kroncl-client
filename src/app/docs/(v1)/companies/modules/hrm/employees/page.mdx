# 1. Сотрудники

Сотрудник — центральная сущность модуля управления персоналом, если не всей системы. В этой документации
(глава <Link href='/docs/companies/accesses' className={styles.link}>«Доступы»</Link>) уже упоминалось основное отличие <span className={styles.underline}>аккаунта на платформе Kroncl</span>
от <span className={styles.underline}>записи сотрудника</span>. Оно заключается в том, что последний не обязан быть зарегистрированным на платформе Kroncl, чтобы попадать в отчётность организации. Такой подход является единственно возможным
в организациях, централизованный учёт в которых ведут менеджеры без личного участия остальных членов коллектива.

Разберём функционал, доступный для работы с этой сущностью.

# 2. Просмотр сотрудников

Просмотр штата доступен в разделе <span className={styles.underline}>«Сотрудники»</span> (левая боковая панель пространства организации).

<MDXImageBlock src='/images/docs/company-employees.png' label='Просмотр сотрудников.' />

Сверху страницы вы увидите разделение модуля на разделы, в том числе <Link href='/docs/companies/modules/hrm/positions'>«Должности»</Link>. Для перехода между ними достаточно нажать на нужную секцию.

Для быстрого поиска по картам сотрудников воспользуйтесь поисковиком (сверху над картами) (<span className={styles.secondary}>предоставляется для всех страниц со списками и коллекциями, вне зависимости от модуля</span>).
Поиск осуществляется по принципу нахождения наиболее релевантного варианта на основе имени, фамилии, отчества, номера телефона, электронной почты или ID сотрудника.

<MDXImageBlock src='/images/docs/company-search-employee.png' label='Поиск сотрудника.' />

# 3. Создание сотрудника

Для создания новой карты сотрудника достаточно нажать на кнопку «Создать» на главной странице просмотра всех существующих карт.

<MDXImageBlock src='/images/docs/company-create-employee.png' label='Создание сотрудника.' />

Некоторые поля, которые может иметь карта:

- имя
- фамилия (опционально)
- рабочий номер (опционально)
- рабочая почта (опционально)
<br />

После создания записи вы попадёте на страницу просмотра данных сотрудника.

# 4. Просмотр карты

Просмотр карты сотрудника доступен сразу после её создания либо при клике на карту на странице просмотра штата сотрудников. 
На странице сотрудника доступны действия по редактированию личных данных, деактивации и активации.

<MDXImageBlock src='/images/docs/company-employee.png' label='Просмотр карты сотрудника.' />

# 5. Привязка аккаунта

На странице просмотра информации о сотруднике доступно действие по привязке аккаунта к карте сотрудника. 
В соответствии с иерархией переопределения разрешений (глава <Link href='/docs/companies/accesses' className={styles.link}>«Доступы»</Link>) указанному аккаунту будут присвоены
все разрешения от должности карты сотрудника (при её наличии).

Обращаем внимание, что даже если аккаунт уже привязан к другому сотруднику, система разрешит переопределить эту связь.

<MDXImageBlock src='/images/docs/company-change-account-employee.png' label='Привязка аккаунта.' />

# 6. Привязка к должности

Привязка сотрудника к должности доступна в секции должности на странице карты сотрудника.

# 7. Деактивация сотрудника

Для деактивации карты сотрудника достаточно нажать на кнопку «Деактивировать» на странице просмотра информации о сотруднике.
После подтверждения карта не будет безвозвратно удалена, но последующие действия с ней (например, назначение деактивированного сотрудника ответственным
за операцию в финансовом модуле) будут недоступны.

Владельцы организации в любой момент могут снова активировать карту сотрудника.

<MDXImageBlock src='/images/docs/company-deactivate-employee.png' label='Деактивация сотрудника.' />
