# 1. Клиенты

<span className={styles.underline}>«Клиент»</span> — центральная сущность модуля клиентской базы. Запись клиента переиспользуется во множестве мест — от аналитики трафика до 
привязки к <Link href='/docs/companies/modules/dm' className={styles.link}>сделкам</Link>.

# 2. Просмотр клиентов

Для просмотра созданных клиентов перейдите в раздел <span className={styles.underline}>«Клиенты»</span> (боковая панель пространства организации). Для просмотра конкретной записи
нажмите на соответствующую кнопку в карточке клиента.

<MDXImageBlock src='/images/docs/company-crm-clients.png' label='Просмотр клиентов.' />

# 3. Создание клиента

Для инициализации новой записи необходим ряд обязательных и опциональных полей:

- имя
- тип клиента (физическое/юридическое лицо)
- источник привлечения (<span className={styles.accent}>обязательно</span>)
- фамилия (опционально)
- отчество (опционально)
- номер телефона (опционально)
- почта (опционально)
- комментарий (опционально)

<MDXImageBlock src='/images/docs/company-crm-create-client.png' label='Создание нового клиента.' />

Обязательность привязки нового клиента к созданному ресурсу привлечения гарантирует отсутствие "потерянных" клиентов в клиентской базе организации;
анализировать такой трафик гораздо легче.

<MDXImageBlock src='/images/docs/company-crm-create-client-cut.png' label='Выбор ресурса привлечения.' />

# 4. Просмотр клиента

После создания записи клиента доступна страница просмотра и управления, включая <span className={styles.accent}>деактивацию/активацию</span> и <span className={styles.accent}>редактирование</span>.

<MDXImageBlock src='/images/docs/company-crm-client.png' label='Просмотр клиента.' />

# 5. Деактивация

Удалять клиента напрямую нельзя, для изменения статуса видимости используйте деактивацию. Запись не удалится физически, но станет недоступна для новых операций.

<MDXImageBlock src='/images/docs/company-crm-deactivate-client.png' label='Деактивация клиента.' />
