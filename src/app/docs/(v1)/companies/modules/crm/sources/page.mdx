# 1. Ресурсы привлечения

<span className={styles.underline}>Ресурс привлечения</span> описывает метод приобретения контакта клиента. Kroncl
предоставляет возможность регистрации собственных ресурсов и дальнейшего распределения по ним клиентов.

Например, источником привлечения может быть: Авито, Яндекс.Директ, сайт компании и так далее.

# 2. Просмотр ресурсов

Для просмотра зарегистрированных ресурсов привлечения перейдите в раздел <span className={styles.underline}>«Клиенты»</span> (боковая панель пространства организации), затем откройте вкладку <span className={styles.underline}>«Ресурсы привлечения»</span>. 
На странице просмотра вы сможете фильтровать записи по типу: <span className={styles.accent}>активные</span>, <span className={styles.accent}>неактивные</span>, <span className={styles.accent}>системные</span>, все.

<MDXImageBlock src='/images/docs/company-crm-sources.png' label='Просмотр ресурсов привлечения.' />

# 3. Создание ресурса

Для создания нового ресурса привлечения нажмите на соответствующую кнопку на странице просмотра всех записей.

<MDXImageBlock src='/images/docs/company-crm-create-source.png' label='Создание ресурса привлечения.' />

Для инициализации новой записи необходимо указать ряд обязательных и опциональных полей:

- наименование
- тип (органический, социальные сети, реферальный... - пояснения доступны на странице создания)
- ссылка (URL) (опционально)
- комментарий (опционально)
<br />

# 4. Просмотр ресурса

После создания ресурса будет доступна страница просмотра. Для всех ресурсов доступно:

- <span className={styles.accent}>просмотр клиентов ресурса</span>
- <span className={styles.accent}>редактирование</span>
- <span className={styles.accent}>деактивация/активация</span>

<MDXImageBlock src='/images/docs/company-crm-source.png' label='Просмотр ресурса привлечения.' />

# 5. Деактивация

Удалить уже созданный ресурс привлечения нельзя - это может повлиять на анализ уже созданных клиентов. Kroncl предоставляет
стандартный механизм деактивации/активации записи.

При деактивации запись не удалится физически, но перестанет быть доступна для новых операций.

<MDXImageBlock src='/images/docs/company-crm-deactivate-source.png' label='Деактивация записи.' />
