# 1. Кредиты и Дебеты

Для корректной работы с подсистемой долговых обязательств необходимо понимать разницу между двумя основными понятиями долгового учёта в рамках Kroncl:

- <span className={styles.underline}>Кредит</span> — заём средств у вашей организации другим лицом/организацией;
- <span className={styles.underline}>Дебет</span> — заём средств вашей организацией у других лиц/организаций.

<br />

На разделении на два типа основана вся подсистема управления долговыми обязательствами.

# 2. Просмотр обязательств

Для просмотра базы текущих долговых обязательств перейдите на страницу <span className={styles.underline}>«Финансы»</span> (левая боковая панель пространства организации), затем выберите раздел <span className={styles.underline}>«Долги»</span>, после чего перейдите на вкладку <span className={styles.underline}>«Главное»</span>.

<MDXImageBlock src='/images/docs/company-fm-debts.png' label='Просмотр долговых обязательств.' />

# 3. Регистрация обязательства

Для регистрации долгового обязательства нажмите на соответствующую кнопку действия на странице просмотра существующих обязательств.

Каждая запись, вне зависимости от типа (кредит/дебет), имеет ряд обязательных и опциональных полей:

- Тип (кредит/дебет)
- Начальная сумма займа
- Валюта (по умолчанию российский рубль)
- Годовая процентная ставка (займ может быть беспроцентным — ставка = 0)
- Старт и конец периода займа
- Контрагент
- Наименование (опционально)
- Комментарий (опционально)

<MDXImageBlock src='/images/docs/company-fm-debts-create.png' label='Регистрация обязательства.' />

# 4. Карта займа

Разберём карту предпросмотра созданного обязательства (отображается на странице просмотра всех существующих обязательств).

Заметим ключевые поля:

- Тело займа — изначальная сумма, указанная при создании займа;
- Идентификатор периода — нижняя область карты разбита на секторы, часть из которых заполнена акцентным цветом. Идентификаторы показывают прошедший период от начала займа до его окончания. Чем больше секторов заполнены цветом, тем меньше времени до окончания займа;
- Итог на окончание периода — итоговая сумма с учетом годовых процентов.

<MDXImageBlock src='/images/docs/company-fm-debts-card.png' label='Карта долгового обязательства.' />

# 5. Просмотр займа

Для просмотра полной информации о займе нажмите на соответствующую кнопку в карточке займа.

На странице просмотра займа доступны следующие действия:

- Создание новой операции по займу
- Просмотр операций по займу
- Деактивация/активация займа

<br />

Kroncl не предоставляет возможности редактирования существующих долговых обязательств и связанных операций. Это осознанное архитектурное решение (на первый взгляд кажущееся неудобством), но являющееся единственным верным способом предотвратить финансовый аудит предприятия в сюр.

{/* <MDXImageBlock src='/images/docs/company-fm-debts-view.png' label='Просмотр займа.' /> */}

# 6. Операции займа

Каждый займ предоставляет возможность привязки соответствующих финансовых операций — например, постепенные выплаты дебета, распределённого на несколько транзакций.

На странице займа отображаются все финансовые операции, связанные с данным займом, а также предоставляется возможность создания новой операции.

Важно: все операции, связанные с займами, распределяются между соответствующими категориями (подробнее в разделе <Link className={styles.link} href='/docs/companies/modules/fm/movement-of-funds/categories'>«Категории»</Link>) — доход по кредитам и расход по займам (системные категории). Это позволяет видеть все операции по займам на страницах этих категорий. Кроме того, на странице анализа (подробнее в разделе <Link className={styles.link} href='/docs/companies/modules/fm/movement-of-funds/analysis'>«Анализ операций»</Link>) финансовых операций все созданные в процессе выплат займов операции попадают в аналитику разделения по категориям, что, в свою очередь, позволяет видеть отношение кредитов к дебетам и к другим категориям финансового оборота компании.

{/* <MDXImageBlock src='/images/docs/company-fm-debts-operations.png' label='Операции займа.' /> */}

# 7. Деактивация займа

Деактивация займа используется для закрытия или отмены займа. При деактивации запись займа и все связанные данные (в том числе финансовые операции) физически не удаляются, но сам займ переходит в статус <span className={styles.underline}>«Неактивен»</span>.
