# 1. Контрагенты

<span className={styles.underline}>Контрагент</span> в рамках Kroncl — лицо/организация, берущая/дающая средства в долг (у) вашей компании. Kroncl использует такой формат ведения контрагентов только для подсистемы долговых обязательств.

Вне зависимости от типа контрагента (физическое лицо, банк, другая организация), запись о его существовании хранится в базе контрагентов вашей организации. Без указания контрагента не получится создать запись кредита/дебета.

## 2. Просмотр контрагентов

Для просмотра базы зарегистрированных контрагентов перейдите на страницу <span className={styles.underline}>«Финансы»</span> (левая боковая панель пространства организации), затем выберите раздел <span className={styles.underline}>«Долги»</span>, после чего перейдите на вкладку <span className={styles.underline}>«Контрагенты»</span>.

<MDXImageBlock src='/images/docs/company-fm-debts-counterparties.png' label='Просмотр долговых контрагентов.' />

## 3. Создание контрагента

Для создания контрагента нажмите соответствующую кнопку действия на странице просмотра контрагентов.

Запись контрагента содержит ряд обязательных и опциональных полей:

- Наименование контрагента
- Тип контрагента (банк/организация/физическое лицо)
- Комментарий (опционально)

<br />

<MDXImageBlock src='/images/docs/company-fm-debts-create-counterparty.png' label='Создание долгового контрагента.' />

После создания контрагента вы будете переадресованы на его страницу.

## 4. Просмотр контрагента

Для просмотра информации о контрагенте достаточно нажать на соответствующую кнопку в карточке на странице просмотра всех зарегистрированных контрагентов.

На этой же странице будут доступны действия редактирования, просмотра информации и деактивация/удаление объекта.

## 5. Деактивация/активация

По аналогии с большинством объектов в организации, Kroncl предотвращает <span className={styles.accent}>жёсткое удаление</span> записи контрагента в целях обеспечения консистентности данных долговых обязательств.

Деактивация контрагента физически не удаляет уже сохраненные данные, а лишь скрывает его для создания новых долговых обязательств.
