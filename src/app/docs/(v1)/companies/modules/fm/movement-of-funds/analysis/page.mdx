# 1. Анализ операций

Очевидно, в определенный момент времени ваша организация при исправном ведении финансовой отчетности видит в счетчике операций трех-, четырех- и ... значные числа.
Такой пласт информации о совершенных операциях открывает обширные возможности для анализа финансовой отчетности за целевой промежуток времени.

Анализ (аналитика) операций доступен(-а) в разделе <span className={styles.underline}>«Сквозная аналитика»</span> (модуль <span className={styles.underline}>«Финансы»</span>).

<MDXImageBlock src='/images/docs/company-fm-analysis-open.png' label='Сквозная аналитика.' />

После клика на нужную секцию произойдет загрузка аналитики финансов.

<MDXImageBlock src='/images/docs/company-fm-analysis.png' label='Сквозная аналитика.' />

# 2. Целевой период

Данные во всех вкладках раздела анализа высчитываются исходя из <span className={styles.underline}>целевого периода анализа</span>. По умолчанию 
целевым периодом является <span className={styles.accent}>начало и конец ведения финансовой отчетности</span>.

Для изменения периода анализа (можно менять как обе, так и одну из дат) заполните соответствующие поля в шапке раздела и нажмите на кнопку <span className={styles.underline}>«Применить»</span>.

<MDXImageBlock src='/images/docs/company-fm-analysis-period.png' label='Целевой период анализа.' />

После применения произойдет пересчет данных.

# 3. Главное

Подраздел <span className={styles.underline}>«Главное»</span> аналитики дает представление о базовых счетчиках финансов и выстраивает динамику трат/доходов с промежутком в <span className={styles.underline}>1 день</span>.

<MDXImageBlock src='/images/docs/company-fm-analysis-main.png' label='Аналитика финансов - «Главное».' />

Левая область подраздела посвящена счетчикам, правая (большая) - графику динамики.

## 3.1 Счетчики

В числе счетчиков финансов:

- баланс к концу периода
- общая сумма дохода
- общая сумма расходов
- количество операций за период
- средний чек операции
<br />

Обращаем внимание, что Kroncl <span className={styles.accent}>НЕ считает операции трат отрицательными</span>, высчитывая средний чек за период времени как среднее арифметическое отрицательных (траты) и положительных (доходы). 
Все операции, вне зависимости от направления, имеют положительный знак, вследствие чего и <span className={styles.accent}>средний чек за целевой период будет положительным числом</span>. 

## 3.2 Динамика

На графике динамики каждая точка (день) имеет две характеристики: 

- сумма трат
- сумма доходов
<br />

Таким образом, в течение целевого периода строится <span className={styles.underline}>два графика</span>: динамики трат (красный) и доходов (зеленый). 
При наведении курсора на точку будут видны точные числа двух этих показателей за соответствующий день.

# 5. По категориям

В подразделе <span className={styles.underline}>«По категориям»</span> доступна столбчатая диаграмма распределения трат и доходов по категориям (подробнее в главе 
<Link href='/docs/companies/modules/fm/movement-of-funds/categories' className={styles.link}>«Категории»</Link>).

Внимательный читатель помнит, что каждая категория имеет только одно из двух направлений: либо <span className={styles.underline}>трата</span>, либо <span className={styles.underline}>доход</span>.
Однако на диаграмме распределения у каждой категории есть два столбца для двух направлений.
Не забывайте про <span className={styles.underline}>обратные операции</span> (подробнее в главе <Link href='/docs/companies/modules/fm/movement-of-funds/operations' className={styles.link}>«Операции»</Link>), 
получающие противоположное направление вне зависимости от категории изначальной операции.

Таким образом, если для категории «Фонд Оплаты Труда» (расходная) суммарные траты = <span className={styles.tag}>10 000 &#8381;</span>, а доходы = <span className={styles.tag}>4 000 &#8381;</span> - это будет означать, что
для изначальных операций на сумму <span className={styles.tag}>4 000 &#8381;</span> были созданы обратные операции на ту же сумму. Аналогично для категорий любого направления.
