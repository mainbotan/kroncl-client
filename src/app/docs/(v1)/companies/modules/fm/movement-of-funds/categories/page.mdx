# 1. Категории

Все финансовые операции любой организации имеют классификацию. Одни операции направлены на погашение долговых
обязательств, другие - на выплату зарплат сотрудникам. Чтобы различать цель/источник появления операции, Kroncl предлагает
использовать категории.

# 2. Просмотр категорий

Для просмотра существующих категорий финансовых операций перейдите в раздел <span className={styles.underline}>«Финансы»</span> -> <span className={styles.underline}>«Категории»</span>.

<MDXImageBlock src='/images/docs/company-fm-open-categories.png' label='Категории финансовых операций.' />

После перехода в нужный раздел вы увидите карты категорий с минимально полезной информацией о каждой из них.
На этой же странице доступен поиск по всем существующим категориям и фильтрация по направлению (все/доходные/расходные).

<MDXImageBlock src='/images/docs/company-transactions-categories.png' label='Просмотр категорий.' />

# 3. Системные категории

Перейдя в раздел финансов в первый раз, вы увидите ряд категорий, созданных без вашего участия. Каждая из этих категорий
имеет пометку <span className={styles.accent}>«Системная категория»</span>.

Системные категории нужны для того, чтобы другие модули (как например <Link href='/docs/companies/modules/dm' className={styles.link}>«Управление сделками»</Link>) использовали
для связанных операций заготовленную классификацию. Системные категории, в отличие от созданных организацией, <span className={styles.underline}>нельзя удалить</span>, что 
исключает вероятность того, что внешний модуль попробует создать финансовую операцию с привязкой к несуществующей категории, которую намеренно или случайно удалил сотрудник организации.

Таким образом, Kroncl гарантирует, что все финансовые операции, созданные в результате расчета сделки модуля управления сделками, будут записаны
в категорию <span className={styles.underline}>«Доход от сделок»</span>.

# 4. Создание категории

Для создания категории нажмите на кнопку <span className={styles.underline}>«Новая категория»</span> на странице просмотра существующих категорий.

<MDXImageBlock src='/images/docs/company-transactions-create-category.png' label='Создание категории.' />

Некоторые поля, свойственные категории:

- наименование
- описание (опционально)
- тип (расход/доход)
<br />

После создания категории произойдет перенаправление на ее страницу.

# 5. Просмотр категории

Для просмотра категории нажмите на соответствующую карту на странице просмотра всех существующих категорий. 
Здесь доступны действия по редактированию/удалению категории.

Также на странице категории отображаются финансовые операции с фильтрацией по периоду, привязанные к данной категории.

<MDXImageBlock src='/images/docs/company-transactions-category.png' label='Просмотр категории.' />

# 6. Удаление категории

Для безвозвратного удаления категории нажмите на кнопку <span className={styles.underline}>«Удалить»</span> на странице просмотра 
категории, после чего подтвердите действие.

Удаление категории не приведет к удалению связанных финансовых операций (что невозможно по самой природе финансового аудита), однако, очевидно, 
финансовые операции потеряют принадлежность к удаленной категории.

Напоминаем, что операция удаления недоступна для <span className={styles.underline}>системных категорий</span>.

<MDXImageBlock src='/images/docs/company-transactions-delete-category.png' label='Удаление категории.' />
