# 1. Хранилище компании

В этой документации уже проводилась аналогия организации с изолированным контейнером, 
хранящим данные компании. Мы разделяем два типа хранилищ компании внутри такого контейнера: 
 
- основное хранилище (база данных): отвечает за хранение текстовых данных модулей;
- объектное хранилище: отвечает за хранение документов, изображений и других файлов.
<br />

Каждый тип хранилища в рамках организации имеет свои лимиты, зависящие от выбранного <Link className={styles.link} href='/docs/companies/pricing'>тарифного плана</Link>.
Для удобства читателя мы поместили информацию о лимитах тарифных планов в главу, посвящённую правилам тарификации.

Контур из двух этих подсистем хранения носит обобщённое название <span className={styles.underline}>«Хранилище»</span>.
Информация о хранилище находится в одноимённом разделе (левая боковая панель пространства организации).

<MDXImageBlock src='/images/docs/company-storage.png' label='Просмотр ресурсов хранилищ.' />

# 2. Достижение лимитов

При достижении лимитов хранилища мы автоматически отправим уведомления-алерты на почты владельцев организации, предложив
способы оптимизации хранилища.

В большинстве случаев оптимизация ресурсов хранилища достигается путём очистки <Link className={styles.link} href='/docs/companies/logs'>логов компании</Link>
или перехода на расширенные тарифные планы.

Мы также открыты для внепланового расширения лимитов конкретных организаций, выходящих за
пределы классических тарифных планов. В том числе, команда Kroncl может помочь организации в миграции
на отдельный инстанс хранилища данных исходя из заданных бюджетов.

По вопросам о расширении лимитов обращайтесь в <Link className={styles.link} href='/support'>службу поддержки</Link>.

# 3. Системным администраторам

Ваша организация может иметь в штате сотрудников системных администраторов, отвечающих
за взаимодействие с внешними сервисами. Именно для этой категории пользователей
мы предоставляем расширенную информацию о ресурсах хранилища базы данных.

В том числе:

- размер индексов БД организации
- размер таблиц
- количество таблиц
- количество активных подключений
- количество строк в таблицах БД организации
<br />
Также мы предоставляем подробную классификацию по ресурсам базы данных организации с точностью
до распределения по таблицам модулей.

Сотрудничество с системными администраторами компаний-клиентов помогает нам оптимизировать
архитектуру базы данных. Если вы стали свидетелем снижения производительности работы базы данных, обращайтесь
в <Link href='/support' className={styles.link}>службу поддержки</Link>.
